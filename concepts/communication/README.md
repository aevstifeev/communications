# Communication Concepts

Здесь живут две сущности:

- отдельные концептуальные направления модуля коммуникации;
- параллельные варианты роли созвонов в системе, которые нужно сравнивать между собой.

Текущая структура:

- `meeting-card/` - отдельное направление про коммуникацию в карточке;
- `space-calendar/` - отдельное направление про календарный сценарий;
- `task-integration/` - отдельное направление про интеграцию в задачи;
- `hub.html` - единая точка входа для сравнения вариантов;
- `baseline/` - зафиксированная текущая версия;
- `variant-inline-panel/` - вариант A: созвоны как часть чатов, каналов и текущей коммуникации;
- `variant-thread-workspace/` - вариант B: созвоны как отдельный модуль коммуникации наравне с чатами и календарем;
- `variant-context-drawer/` - вариант C: резервный, сейчас неактивен.

Правило работы:

- текущую рабочую реализацию сначала сохраняем в `baseline/`;
- новые гипотезы не встраиваем сразу в общий `index.html`;
- каждый новый вариант получает свою папку и короткое описание в `docs/communication-variants.md`.
